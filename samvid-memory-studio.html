The core insight

Eight agents.
One brain.

Today's AI tools are powerful but isolated. Cursor doesn't know what you decided in Claude. Claude doesn't know what Windsurf learned last week. Every tool starts from zero, every session.

Samvid solves this by running a local MCP server that any agent can connect to. One database. One memory layer. All your tools share it — simultaneously, in real time.

Add a memory in the desktop app and your coding agent sees it instantly. Have an agent store a decision and every other agent knows it. The knowledge base grows with your work — not against it.

Memory taxonomy

Not all knowledge
is the same.

Most memory systems store everything as undifferentiated text. Samvid distinguishes 16 types — because a rule and a heuristic behave differently, a goal and a plan require different reasoning, and feedback should be weighted differently from facts.

The agent always knows what kind of thing it recalled — not just what it says. This shapes how it reasons, how it weights contradictions, and when it asks for confirmation before acting.

Memories with importance ≥ 0.9 are protected from automatic forgetting. rule type memories are always preserved — agents must never override them.

fact
Verifiable, stable information
"Postgres for payments service"
preference
User or agent preferences
"prefers concise responses"
rule
Hard constraints — always followed, never overridden
"never commit to main"
heuristic
Soft guidelines from experience
"'quick' means one-liner"
goal
Active objective being pursued
"ship v2 by end of quarter"
plan
Concrete steps toward a goal
"step 1: refactor auth…"

"that answer was too long"

Typed associations

A knowledge graph,
not a list of facts.

Memories can be linked to each other with typed relationships. This lets the agent reason over contradictions, trace causality, and understand when a newer decision supersedes an older one.

supports → This memory corroborates that one. Reinforces confidence when reasoning.
contradicts → Flags a conflict. Agent surfaces the tension before acting.
supersedes → A newer decision replaces an older one. The graph stays current.
caused_by → Links outcomes to their causes. Enables root-cause recall.
implies → Logical entailment. Also: prevents, blocks, is_a, part_of, instance_of.

Each association carries a strength score (0.0–1.0) — confidence evolves as the graph grows.

Memory decay

Memories fade.
Important ones don't.

Every memory carries an importance score (0–1) and a decay rate. Effective importance fades exponentially over time — just like biological memory. Old, untouched knowledge naturally fades. Frequently accessed, high-importance memories persist.

This keeps the knowledge base from becoming a graveyard of irrelevant notes. The Forget operation prunes memories whose effective importance has dropped below a threshold — but anything with importance ≥ 0.9 is always protected.

You set the decay rate per memory — or let the system default. TTL is also available for memories you explicitly want to expire: "remember this for one week".

Lifecycle management

A knowledge base that
cleans itself.

Four operations keep your memory store lean, consistent, and useful as it grows. Run them manually, on a schedule, or let an agent trigger them automatically — all support dry-run preview before committing.

🌫 forget
Prune stale memories
Deletes memories whose effective importance (base importance × decay) has dropped below a configurable threshold. Anything with importance ≥ 0.9 is always preserved, regardless of age.
samvid forget --threshold 0.1
🗜 compact
Reclaim disk space
Removes all memories whose TTL has expired, then runs SQLite VACUUM to physically reclaim disk space. Fast and safe — no vector recomputation needed.
samvid compact
🔗 consolidate
Merge near-duplicates
Finds semantically similar memories (cosine distance below a threshold) and merges them into one. The older memory absorbs the newer; the higher importance wins. Supports dry-run so you preview every merge before it happens.
samvid consolidate --dry-run
📝 summarize
Compress memory batches
When a namespace accumulates many raw memories, batch them for LLM summarization. The summary is stored as a new memory, and the source memories' importance is automatically downgraded — keeping the store concise.
samvid summarize-prep --namespace work
Episodic sessions

Conversations you can
replay.

Related memories are grouped into sessions — an episode, a conversation, a project arc. Each memory in a session links to the previous one via previous_memory_id, forming a traversable chain.

The replay_session tool reconstructs the entire session as a formatted chronological transcript — ready to inject directly into an LLM context window. The agent re-reads exactly what happened, in order.

Long content is auto-chunked at sentence boundaries and chained automatically — so even a 50-page document becomes a replayable episodic chain, not a single massive blob.

Privacy & architecture

Single file.
Zero cloud.
Forever yours.

The embedding model is a 542 MB quantized ONNX file that runs entirely on your CPU. Every vector is computed locally — no network call is ever made for memory operations.

The entire database is a single SQLite file in your OS user-data directory. Backup with cp samvid.db backup.db. Migrate to another machine: copy the file and reinstall.

Your memories survive uninstalls. The database lives completely outside the application install path. Reinstalling picks it up automatically.
