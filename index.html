Typed associations

A knowledge graph,
not a list of facts.

Memories link to each other with typed relationships and a strength score. The graph grows as agents reason — supporting, contradicting, superseding, and tracing causes between every fact they store.

supports
Corroborates another memory. Reinforces confidence during reasoning.
contradicts
Flags a conflict. Agent surfaces the tension before acting on either.
supersedes
A newer decision replaces an older one. The graph stays current.
caused_by
Links outcomes to their causes. Enables root-cause recall.
implies
Logical entailment. Also: prevents, blocks, is_a, part_of, instance_of.
Each association carries a strength score (0.0–1.0) — confidence evolves as the graph grows.

Memory decay

Memories fade.
Important ones don't.

Every memory carries an importance score (0–1) and a decay rate. Effective importance fades exponentially over time — just like biological memory. Old, untouched knowledge naturally fades. Frequently accessed, high-importance memories persist.

This keeps the knowledge base from becoming a graveyard of irrelevant notes. The forget operation prunes memories whose effective importance has dropped below a threshold — but anything with importance ≥ 0.9 is always protected.

You set the decay rate per memory — or let the system default. TTL is also available for memories you explicitly want to expire: "remember this for one week".

Lifecycle management

A knowledge base that
cleans itself.

Four operations keep your memory store lean, consistent, and useful as it grows. Run them manually, on a schedule, or let an agent trigger them automatically — all support dry-run preview before committing.

⌬forget

Prune stale memories

Deletes memories whose effective importance (base importance × decay) has dropped below a configurable threshold. Anything with importance ≥ 0.9 is always preserved, regardless of age.

samvid forget --threshold 0.1
⤓compact

Reclaim disk space

Removes all memories whose TTL has expired, then runs SQLite VACUUM to physically reclaim disk space. Fast and safe — no vector recomputation needed.

samvid compact
↬consolidate

Merge near-duplicates

Finds semantically similar memories (cosine distance below a threshold) and merges them into one. The older memory absorbs the newer; the higher importance wins. Supports dry-run so you preview every merge before it happens.

samvid consolidate --dry-run
∑summarize

Compress memory batches

When a namespace accumulates many raw memories, batch them for LLM summarization. The summary is stored as a new memory, and the source memories' importance is automatically downgraded — keeping the store concise.

samvid summarize-prep --namespace work
Episodic sessions

Conversations you can
replay.

Related memories are grouped into sessions — an episode, a conversation, a project arc. Each memory in a session links to the previous one via previous_memory_id, forming a traversable chain.

The replay_session tool reconstructs the entire session as a formatted chronological transcript — ready to inject directly into an LLM context window. The agent re-reads exactly what happened, in order.

Long content is auto-chunked at sentence boundaries and chained automatically — so even a 50-page document becomes a replayable episodic chain, not a single massive blob.

Questions

Quick answers

Does my data ever leave my computer?

No. Samvid Memory Studio runs entirely on your machine. The database is a local SQLite file. There are no accounts, no telemetry, and no cloud sync.

Which AI tools work with Samvid?

Any tool that supports the Model Context Protocol (MCP). That includes Claude Desktop, Cursor, VS Code, Windsurf, OpenCode, Codex, and more.

Do I need Python or any technical setup?

No. The desktop app is a single installer with everything bundled. Python, the embedding model, and all dependencies are included.

What happens to my data if I uninstall?

Your memory database lives in a separate system directory that survives uninstall. Reinstalling picks up right where you left off. You can also export to JSON anytime.
